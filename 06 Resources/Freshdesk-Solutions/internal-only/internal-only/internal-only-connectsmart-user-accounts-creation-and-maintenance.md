# *|Internal Only| - ConnectSmart User Accounts Creation and Maintenance:

This article outlines procedures around customer requests for User Accounts for ConnectSmart Host or the QSR Portal, including account creation, removal, User Role assignment, and User Role permissions.


 


At the time of Company creation, Data Management will create an initial User Account according to details provided in the Enterprise Configuration Worksheet we receive from the customer with the order.  This is the only instance in which Data Management is permitted to grant access.  At that time, Data Management provides one of the disclaimers below depending on the recipient:


- **Account Managers/Channel Mangers/Resellers:** _Please note that this user will be the System Admin login for the Company. The System Admin login is used to access the QSR Portal, create additional Users, and grant/restrict User access as needed. To preserve the security of our customers' accounts, QSR's agents are not permitted to add additional User accounts or alter their access/permission._

- **End Users:** _Please note that this user will be the System Admin login for your Company. The System Admin login is used to access the QSR Portal, create additional Users, and grant/restrict User access as needed. To preserve the security of our customers' accounts, QSR's agents are not permitted to add additional User accounts or alter their access/permission._


 


QSR Account Managers and Support agents are **not** permitted to grant User access or adjust User Roles/Permissions under any circumstances. 


 


See the information below to determine how to handle an access request for a customer:


 


**Large Direct Customers (Darden, YUM!, Disney, etc):**


- User access should be managed by their own IT department..

- Store level personnel should be directed to their IT department.

- IT personnel can be walked through the process of creating a User/adjusting the User Role/adjusting Role permissions, but we cannot make the changes for them.

- If the IT agent does not have access to create/edit Users or User Roles, you can provide them with names of Users within their company who do have access.

- If existing Users with permissions to manage User Accounts are no longer with the company (or otherwise unavailable), the Technical Projects team can create a User Account for them if they can verify the Name and last 4 digits of the credit card we have on file. 


 


**Reseller Accounts:**


- Reseller accounts may manage their own User access, or rely on their reseller to manage it for them.

- Store level personnel should be directed to either their IT department or their reseller support (depending on their own internal processes).

- Restaurant IT personnel should be directed to the reseller.

- Each reseller should have a “Reseller Login” that they can use to create/edit User accounts for the Companies they service.  If a Reseller believes they do not have a Reseller login, they can be directed to their channel manager to request one.

- Reseller personnel can be walked through the process of creating a User/adjusting the User Role/adjusting Role permissions, but we cannot make the changes for them.


 


**Smaller Direct Customers (usually Host Standalone/IOS/”Cloud” customers):**


- These are direct customers who do not have an internal IT department.

- The caller can be walked through the process of creating a User/adjusting the User Role/adjusting Role permissions, but we cannot make the changes for them.

- If the caller does not have access to create/edit Users or User Roles, you can provide them with names of Users within their company who do have access.

- If existing Users with permissions to manage User Accounts are no longer with the company (or otherwise unavailable), the Technical Projects team can create a login if they can verify the Name and last 4 digits of the credit card we have on file.


 


These policies are designed to protect the security of our customer’s accounts.  If you have any questions, please reach out to [technicalprojects@qsrautomations.com] or the #Help_Support Slack channel.
