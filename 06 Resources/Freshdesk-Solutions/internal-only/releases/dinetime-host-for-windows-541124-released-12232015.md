# *DineTime Host for Windows 5.4.112.4  (Released 12/23/2015)

**New Features/Enhancements**** **

- Parties on hold are excluded from the quote calculations **Request: GAR, Cracker Barrel**

- Added the Party Mix dialog to the transfer table UI

- Added the Reservation Confirmation number to the Edit Reservation form

- Added support for Server shifts

- Prompts user to assign a server when opening a table with no server assigned **Request: Darden**

- Changed several lists to use natural sort

- Added a button to allow walking away a seated party when allowed by settings

- The add shift dialog was changed to a wizard that allows the user to create a new or copy an existing shift **Request: KRM, Cracker Barrel**

- Sorted lists retain the last sort unless configured to revert (Configuration UI targeted for Portal 5.1 date TBD.)

- Validates email address format when entered

- Adds DineTime pager Alerts and overall Alerts framework (Only pager alerts are configurable at this time. Future alerts enhancements are planned in releases TBD)

- Encrypts any stored Allergy data  (Also requires Enterprise changes target date 12/17 and Host 5.2.153.1 target date TBD) **Request: Disney**

- Uses default filename for database backup when not specified

- Updates SQL Express installation to 2012 SP2 for Windows 10 compatibility
**Bug Fixes**** **

- Fixes issue with table name being sent in an inconsistent fashion** Reported: Granite City**

- Fixes issues caused when DineTime server receives NULL SMS messages **Reported: 716 Food and Sport, NYAJ's Bethseda  (001-00-051169) (001-00-051146)**

- Fixes an issue with the recalled 5.4.112.0 general release related to Open/Dirty tables, EOD, and alerts

- Fixes performance issues with high alert volumes
