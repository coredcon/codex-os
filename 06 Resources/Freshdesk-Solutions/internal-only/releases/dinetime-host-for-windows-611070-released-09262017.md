# *DineTime Host for Windows 6.1.107.0  (Released 09/26/2017)

**New Features/Enhancements** 

- Supports GuestView media stored locally  **Jira#: **[**GM-150**]

- Supports popup alerts with timed and snooze features **Jira#: **[**GM-390 - 391, 403 - 405, 447, 448**]** Request: BJ’s (**[**DE-461**] **/** [**GM-382**]**) **_Note: Feature will be supported in Portal 5.11.132+, Host (iPad) 4.2+_

- Displays visit location indicator when guest is close to the restaurant in Server sites **Jira#: **[**GM-463 – 466**]** (**[**GM-462**]**) **_Note: Feature will be supported in Host (iPad) 4.2+, Assist 2.1+_

- Logs the Host Client’s IP address on startup of client  **Jira#: **[**GM-456**]** Request: Internal (**[**DE-537**]** / **[**117546**]**)**

- Fixes issue where double assignment of reservations to tables was allowed  **Jira#: **[**GM-408**]** Reported: CST/Bella Italia/Mancy’s (**[**DE-366**]** / **[**114595**]**)**
** ****Bug Fixes** 

- Fixes issue with displaying YouTube content in GuestView  **Jira#: **[**GM-277**]

- Fixes issue with SyncSource  **Jira#: **[**GM-513**] **Reported: Winston’s Grille (**[**DE-364**]** / **[**114595**]**) **_Note: This temporarily reverts the Reservation Visit Notes functionality to how it behaved previously, which will resolve Winston’s reported issue. Long term fix to Visit Note truncation is targeted for later this year_
