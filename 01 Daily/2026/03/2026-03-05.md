## Vox Session Digest (Session 3)
> Written by Vox at session end. Do not edit manually.

### Context
Deep-dive session on QMD performance. Started with a live comparison of keyword grep vs. semantic search to understand what QMD actually does. Discovered QMD was running on CPU with 0 math cores and no GPU. Spent the session hunting down and fixing a chain of three separate root causes to get full GPU-accelerated semantic search working on the GTX 1060.

### Decisions
- `qmd query` is the correct command (not `qmd search`) — hybrid search with expansion + reranking
- Repair script created at `.claude/scripts/fix-qmd-vulkan.sh` — run after any QMD npm update
- Q4_K_M reranker models don't support the ranking API in node-llama-cpp b8179 — stick with Q8_0
- VRAM management: generate (expansion) model unloads before reranker loads; they can't coexist in 3GB

### Facts Learned
- GTX 1060 3GB has Vulkan runtime but no Vulkan SDK — prebuilt binary (`@node-llama-cpp/win-x64-vulkan@3.17.1`) avoids compile
- QMD's GPU preference was hardcoded CUDA-first; CUDA toolkit absent → infinite build loop
- All 3 QMD models can't fit in 3GB VRAM simultaneously: expand=1107MB, embed=308MB, rerank=606MB
- VRAM fragmentation causes reranker to use ~1241MB when loaded third (vs 606MB alone)
- Fixing load order (unload expand before rerank) gives reranker clean contiguous VRAM
- `qmd query` proven working: "cardiovascular problems medical history" → 90% Health Overview (zero keyword overlap)
