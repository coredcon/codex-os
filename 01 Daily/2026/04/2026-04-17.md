## Vox Session Digest (Afternoon/Evening — Crash Recovery)

### Context
Session started as a crash recovery. Vox restarted watchers and crons, recovered state from WORKING.md. Primary work was a deep HookSnap debugging session triggered by "Failed to Create Endpoint" on the internal testing build. Traced and fixed 4 layered bugs in the mobile app and backend. By end of session: core HookSnap flow fully validated (create, receive, push notification, delete). Version code 5 AAB built and ready to upload to Play Store.

### Decisions
- Server-side IP-based endpoint cap removed — unreliable on mobile/carrier NAT. Free tier cap moved to client-side.
- EAS Update `channel: "production"` added to eas.json — the "production" channel didn't exist, which is why all OTA pushes silently failed.
- Debug endpoint `/api/debug` added temporarily — should remove before public launch.
- HookSnap v5 AAB: `tJckGmmPNRnr88Uo7JC6Fn.aab` — upload to Play Store internal testing next session.

### Facts Learned
- OTA was never working: EAS `production` channel didn't exist until explicitly created via `channel` in eas.json.
- `GET /api/endpoints/[slug]` strips ownerToken server-side — mobile must load it from AsyncStorage, not from the server response.
- Mobile `deleteEndpoint(slug)` was called without ownerToken — server DELETE returned 401 silently, endpoint stayed in Redis.
- IP-based free tier tracking is fundamentally broken on mobile (carrier NAT, proxy IPs, Vercel routing).
- TXRH tickets #452101 and #432796 both updated today (escalation + API performance).
